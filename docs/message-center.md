# 消息中心 (Message Center)

平台统一消息服务，支持站内消息、邮件、短信、企业 IM 推送等多渠道消息触达，提供消息模板、订阅偏好、已读状态管理。

## 消息渠道

| 渠道 | 说明 | 实时性 | 适用场景 |
|------|------|--------|---------|
| 站内消息 | 平台内消息中心 | 实时 (WebSocket) | 日常通知、审批提醒、系统公告 |
| 邮件 | SMTP 外发邮件 | 分钟级 | 正式通知、报表推送、密码重置 |
| 短信 | 短信网关 | 秒级 | 验证码、紧急告警 |
| 企业微信 | 企业微信应用消息 | 实时 | 审批通知、工作提醒 |
| 钉钉 | 钉钉工作通知 | 实时 | 审批通知、工作提醒 |
| 飞书 | 飞书应用消息 | 实时 | 审批通知、工作提醒 |

### 渠道适配器接口

```typescript
interface MessageChannelAdapter {
  readonly type: MessageType;

  /** 发送消息 */
  send(message: MessagePayload): Promise<SendResult>;

  /** 批量发送 */
  batchSend(messages: MessagePayload[]): Promise<BatchSendResult>;

  /** 查询发送状态（异步渠道如短信） */
  queryStatus?(messageId: string): Promise<DeliveryStatus>;
}

interface MessagePayload {
  to: string | string[];          // 接收人（用户ID/手机号/邮箱）
  subject?: string;               // 标题（邮件/站内）
  content: string;                // 内容（支持富文本/HTML）
  contentType: 'text' | 'html' | 'markdown';
  extras?: Record<string, any>;   // 渠道特有参数
}
```

## 消息模板

### 模板定义

```jsonc
{
  "templateId": "tpl_workflow_approval",
  "name": "审批通知",
  "category": "workflow",
  "channels": ["site", "wecom", "email"],
  "variables": ["applicantName", "workflowName", "nodeName", "url"],
  "contents": {
    "site": {
      "title": "{{applicantName}} 提交了审批",
      "content": "流程「{{workflowName}}」已到达「{{nodeName}}」节点，请及时处理。",
      "actionUrl": "{{url}}"
    },
    "email": {
      "subject": "【审批通知】{{workflowName}} 待您审批",
      "content": "<p>{{applicantName}} 提交的流程「{{workflowName}}」已到达「{{nodeName}}」节点。</p><p><a href='{{url}}'>点击处理</a></p>",
      "contentType": "html"
    },
    "wecom": {
      "content": "{{applicantName}} 提交的「{{workflowName}}」待您审批",
      "actionUrl": "{{url}}"
    }
  }
}
```

### 模板管理能力

| 能力 | 说明 |
|------|------|
| 可视化编辑 | 富文本编辑器，支持变量插值 `{{variable}}` |
| 多渠道适配 | 同一模板定义各渠道内容，自动适配 |
| 模板分类 | 系统通知、审批通知、安全告警、营销推广等 |
| 版本管理 | 模板修改保留历史版本，支持回滚 |
| 预览测试 | 支持填写变量值后预览效果，发送测试消息 |


### 用户偏好设置

```jsonc
{
  "userId": "user_001",
  "preferences": {
    "workflow.approve": {
      "channels": ["site", "wecom"],
      "quietHours": { "start": "22:00", "end": "08:00" }
    },
    "workflow.reject": {
      "channels": ["site", "wecom", "email"],
      "urgent": true  // 紧急事件忽略免打扰
    },
    "data.export": {
      "channels": ["site"]
    },
    "system.alert": {
      "channels": ["site", "sms", "wecom"],
      "urgent": true
    }
  },
  "globalSettings": {
    "quietHours": { "start": "22:00", "end": "08:00" },
    "language": "zh-CN"
  }
}
```

### 订阅规则

| 规则类型 | 说明 |
|---------|------|
| 用户自定义 | 用户在消息设置页自行配置每个事件类型的接收渠道 |
| 管理员预设 | 平台/应用管理员预设默认订阅规则，新用户自动继承 |
| 强制订阅 | 安全告警、系统公告等关键事件强制推送，用户不可关闭 |
| 角色订阅 | 按角色批量配置（如"所有审批人默认订阅审批通知"） |

## 消息状态管理

### 站内消息数据模型

```sql
CREATE TABLE messages (
  id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  tenant_id       VARCHAR(64)   NOT NULL,
  recipient_id    VARCHAR(64)   NOT NULL COMMENT '接收人ID',
  sender_id       VARCHAR(64)   NULL COMMENT '发送人ID（系统消息为NULL）',
  template_id     VARCHAR(64)   NULL COMMENT '消息模板ID',
  category        VARCHAR(32)   NOT NULL COMMENT '消息分类',
  title           VARCHAR(256)  NOT NULL,
  content         TEXT          NOT NULL,
  content_type    VARCHAR(16)   NOT NULL DEFAULT 'text',
  action_url      VARCHAR(512)  NULL COMMENT '操作链接',
  is_read         TINYINT(1)    NOT NULL DEFAULT 0,
  read_at         DATETIME      NULL,
  channel         VARCHAR(16)   NOT NULL COMMENT '发送渠道',
  status          ENUM('pending','sent','delivered','failed') NOT NULL,
  related_type    VARCHAR(32)   NULL COMMENT '关联资源类型',
  related_id      VARCHAR(128)  NULL COMMENT '关联资源ID',
  created_at      DATETIME(3)   NOT NULL DEFAULT CURRENT_TIMESTAMP(3),

  INDEX idx_recipient (tenant_id, recipient_id, is_read, created_at),
  INDEX idx_category (tenant_id, recipient_id, category),
  INDEX idx_related (related_type, related_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 消息状态流转

```
消息产生 → pending → sent → delivered
                       └──→ failed (可重试)

已读状态：
unread (is_read=0) → read (is_read=1, read_at=timestamp)
```

### 已读/未读管理

| 能力 | 说明 |
|------|------|
| 单条已读 | 点击消息详情自动标记已读 |
| 全部已读 | 一键标记某分类或全部消息为已读 |
| 未读计数 | 按分类统计未读数量，实时更新（WebSocket 推送） |
| 未读红点 | 顶部导航栏显示未读消息数量 |

## 系统公告

独立于普通消息的公告管理，面向全平台/全租户/全应用的通知：

| 字段 | 说明 |
|------|------|
| `scope` | 公告范围：`platform`（全平台）/ `tenant`（指定租户）/ `app`（指定应用） |
| `level` | 级别：`info` / `warning` / `urgent` |
| `publishAt` | 定时发布时间 |
| `expireAt` | 过期时间（过期后不再展示） |
| `acknowledge` | 是否需要确认（紧急公告可要求用户确认已读） |

## 与现有模块的关系

| 模块 | 关系 |
|------|------|
| **流程引擎** | 通知节点通过消息中心分发，支持多渠道配置 |
| **审计日志** | 安全告警消息（异常登录、权限变更）自动生成 |
| **组织架构集成** | 员工离职/部门变更时自动发送通知 |
| **权限引擎** | 消息模板管理独立权限控制 |
| **多租户** | 消息数据自动注入 `tenant_id`，系统公告支持按租户范围下发 |

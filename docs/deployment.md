# 多租户与部署模式

平台支持 **SaaS 多租户** 和 **私有化部署** 两种模式，通过统一架构适配不同客户需求。

## SaaS 多租户模式

面向云托管场景，多个租户共享同一套基础设施，通过逻辑隔离保障数据安全。

```
┌───────────────────────────────────────────────────┐
│                  SaaS 平台入口                      │
├──────────┬──────────┬──────────┬──────────────────┤
│  租户 A   │  租户 B   │  租户 C   │  ...             │
│  独立配置  │  独立配置  │  独立配置  │                  │
├──────────┴──────────┴──────────┴──────────────────┤
│              共享应用层 + 数据隔离层                   │
├───────────────────────────────────────────────────┤
│              共享基础设施 (计算/存储/网络)              │
└───────────────────────────────────────────────────┘
```

### 多租户核心能力

| 能力 | 说明 |
|------|------|
| 租户隔离 | 支持 **共享数据库+Schema隔离** 和 **独立数据库** 两种模式，按需选择 |
| 租户管理 | 租户开通/冻结/注销，租户管理员自助管理 |
| 配额管理 | 按租户限制用户数、应用数、存储空间、API 调用量等 |
| 租户级配置 | 每个租户可独立配置品牌 Logo、主题色、域名、组件库等 |
| 计量计费 | 按用量/按套餐计费，支持预付费与后付费模式 |
| 资源弹性 | 基于租户负载自动扩缩容，保障 SLA |

### 多租户数据架构

```typescript
// 所有数据模型自动注入租户上下文
interface TenantContext {
  tenantId: string;       // 租户唯一标识
  plan: 'free' | 'pro' | 'enterprise';  // 套餐类型
  quotas: TenantQuotas;   // 资源配额
  config: TenantConfig;   // 租户级配置
}

// 数据库层面自动追加租户过滤
// SELECT * FROM app_data WHERE tenant_id = ? AND ...
```

## 私有化部署模式

面向企业自建场景，客户在自有基础设施上独立部署，数据完全自主可控。

### 部署方式

| 部署方式 | 说明 | 适用场景 |
|---------|------|---------|
| Docker Compose | 单机/少量节点快速部署 | 中小企业、POC 验证 |
| Kubernetes (K8s) | 容器化集群部署 | 大型企业、高可用要求 |
| 离线安装包 | 无外网环境下的离线部署 | 内网/安全合规要求高的场景 |

### 私有化架构

```
┌─────────────────────────────────────────────────────┐
│                  企业内网 / 专属云                      │
│                                                     │
│  ┌───────────┐  ┌───────────┐  ┌─────────────────┐  │
│  │ 应用服务    │  │ 流程服务    │  │ 数据/运算服务    │  │
│  └─────┬─────┘  └─────┬─────┘  └───────┬─────────┘  │
│        │              │                │             │
│  ┌─────┴──────────────┴────────────────┴─────────┐  │
│  │              私有数据库 (MySQL/PostgreSQL)       │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │         企业 SSO / LDAP / AD 集成              │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 私有化核心特性

- **数据自主**：所有数据存储在客户自有数据库，无数据外泄风险
- **SSO 集成**：支持 LDAP / Active Directory / OAuth2 / SAML 2.0 对接企业统一认证
- **运维可观测**：内置 Prometheus + Grafana 监控面板，支持日志审计
- **版本升级**：支持灰度升级与回滚，提供升级迁移脚本
- **定制扩展**：支持私有插件开发，可扩展组件库、流程节点、数据源等

## 双模式架构对比

| 维度 | SaaS 模式 | 私有化部署 |
|------|----------|-----------|
| 部署位置 | 平台云 | 客户自有基础设施 |
| 数据归属 | 平台托管，租户隔离 | 客户完全自主 |
| 多租户 | 原生支持 | 单租户 / 可选多租户 |
| 认证方式 | 平台账号 / OAuth | 企业 SSO / LDAP |
| 运维责任 | 平台方 | 客户方（可选托管运维） |
| 升级方式 | 平台统一升级 | 客户自主升级 |
| 定制能力 | 有限定制 | 深度定制 |
| 成本模型 | 按需订阅 | 永久授权 + 运维成本 |

# 多租户架构

SaaS 多租户模式，多个租户共享同一套基础设施，通过物理隔离（每租户独立 SQLite 文件）保障数据安全。

## 架构概览

```
┌───────────────────────────────────────────────────┐
│                  SaaS 平台入口                      │
├──────────┬──────────┬──────────┬──────────────────┤
│  租户 A   │  租户 B   │  租户 C   │  ...             │
│  独立配置  │  独立配置  │  独立配置  │                  │
├──────────┴──────────┴──────────┴──────────────────┤
│              共享应用层 + 数据隔离层                   │
├───────────────────────────────────────────────────┤
│              共享基础设施 (计算/存储/网络)              │
└───────────────────────────────────────────────────┘
```

## 核心能力

| 能力 | 说明 |
|------|------|
| 租户隔离 | 每租户独立 SQLite 文件（`tenant_{id}.db`），数据物理隔离 |
| 租户管理 | 租户开通/冻结/注销，租户管理员自助管理 |
| 配额管理 | 按套餐限制用户数、应用数、存储空间、API 调用量等 |
| 租户级配置 | 每个租户可独立配置字典、角色、权限、组织架构 |
| 计量计费 | 按用量/按套餐计费，支持预付费与后付费模式 |

## 数据架构

```typescript
// 租户上下文
interface TenantContext {
  tenantId: string;       // 租户唯一标识
  plan: 'free' | 'pro' | 'enterprise';  // 套餐类型
  quotas: TenantQuotas;   // 资源配额
  config: TenantConfig;   // 租户级配置
}

// 数据库文件结构
// data/
//   _system.db              ← 系统级（租户注册、平台管理员、全局字典）
//   tenant_shansui.db       ← 山水集团
//   tenant_acme.db          ← ACME 公司
//   ...
```

## 套餐设计

| 套餐 | 用户数 | 应用数 | 存储 | API 调用/天 |
|------|--------|--------|------|------------|
| 免费版 | 10 | 3 | 1 GB | 1,000 |
| 专业版 | 100 | 20 | 10 GB | 50,000 |
| 企业版 | 不限 | 不限 | 100 GB | 500,000 |
